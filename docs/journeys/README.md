# Journey maps — F5

**Status:** F5 (2026-05-07). 6 representative journey files (selected from ~30 populated cells) + 4 cross-cutting docs. Remaining cells inherit patterns from these representatives; per-cell journey files are written as design-partner conversations begin.

## Why these journeys

F3 articulated *who* personas are. F4 enumerated *what* Liv does. F5 maps **time** — the lifecycle of a human persona inside a configuration, from first-touch through year-one. Without it, every onboarding decision and every lifecycle moment gets invented per-feature, and the product feels fragmented even when each surface is good.

## Selected representative cells (the 6 deepest journeys)

1. **`p2b-solo-barbershop.md`** — P2b Owner-no-Mgr × C2-C4 × Hair (the R5 cell, the deepest bet).
2. **`p1-multi-shop-chain.md`** — P1 Founder × C7 × Hair (chain Founder).
3. **`p2a-single-shop.md`** — P2a Owner-with-Mgr × C5 × Hair (heartland).
4. **`p3-manager.md`** — P3 Manager × C5 × Hair (the staff-side journey).
5. **`p7-customer-regular.md`** — P7 Customer (CT2 Regular) × Hair (the customer journey).
6. **`p2b-chair-rental-host.md`** — P2b Host × C10 × Hair (the under-served wedge cell).

## Cross-cutting docs

- **`onboarding-paths.md`** — configuration-specific + vertical-specific onboarding flows; tagged by starting tech sophistication.
- **`configuration-graduation.md`** — what happens when a salon changes shape.
- **`lifecycle-moments.md`** — universal moments (anniversary, milestones, staff promotion, year-end).
- **`seasonal-and-stage.md`** — calendar overlay (wedding season, Christmas, summer holidays, January fitness surge) and business-stage overlay (pre-revenue / scaling / mature / declining).

## Structure of each journey file

- The cell.
- First touch.
- Sign-up & onboarding.
- First day with Liv.
- First week.
- First month.
- First quarter.
- First year.
- Lifecycle moments to design for.

## Out of frame for v1

- v2 vertical journeys (Wellness, Fitness, Body art) — same structure when those verticals open.
- v3 vertical journeys (Medspa, Allied health) — same structure.
- v2 configuration journeys (mid+ chain, franchise) — same structure.
