# v1.5 scope — heartland + chair-rental + multi-brand

**Status:** Lock-soft (2026-05-07). Target ship: 2027 H1. Acceptance: v1 → v1.5 graduation criteria in `v1-scope.md` + the additions below.

## v1.5 promise (one sentence)

The wedge expanded to cover **chair-rental hosts (C10)**, **small chain founders (C7)**, **single-shop mature with Senior-w-admin at full scope (C6)**, **multi-brand portfolio (C13)**, and the **first cross-tenant intelligence insights** for tenants in formed peer-sets.

## What's added in v1.5

### Personas added or deepened
- **P2b Host (chair-rental)** — Rung 3 baseline, Rung 4 by year-end of v1.5. New cell.
- **P1 Founder × C7 small chain** — Rung 3 → Rung 4 (cross-shop rollup; staff-borrow auto-mediation; chain-level briefing).
- **P4 Senior-w-admin** — graduates from R2 to R3 within scope (auto-handles routine refunds + scoped time-off in their team without escalation).
- **P2a × C13 multi-brand** — new variant of Owner persona; brand-wall scope.

### Configurations added
- **C6 Single-shop mature with sr-w-admin** at deep support.
- **C7 Small multi-shop chain (2–4 shops).**
- **C10 Chair-rental host** — first-class data model, dashboard, automation.
- **C13 Multi-brand portfolio** (≤3 brand-shells).

### Verticals at v1.5
- **Hair (continued).**
- **Beauty** opens (lash, brow, nail, facial). Distinct vertical templates, not "Hair with different services."

### Locales at v1.5
- **English-IE (continued).**
- **English-UK voice** (English-UK accent, RP + Estuary cast). Triggered by UK design-partner cohort.

### Workflows added
- `staff-borrow` — cross-shop staff coverage in C7 chains.
- `chain-rollup` — Sunday digest aggregated across shops for the Founder.
- `host-rent-collect` — automated rent calculation + reminder for C10 hosts.
- `host-renter-dispute-mediation` — first-line dispute handling.
- `multi-brand-briefing` — per-brand Sunday digest plus portfolio rollup for C13.
- `partner-vote` — first-class C12 Partnership decision workflow (deferred from v1).
- `hire` — full hiring intake workflow (job-post draft → application gather → screening DM → interview slots → offer letter draft).

### Features added
- **`chair-rental-dashboard`** — host's surface for renters, rent, occupancy, dispute queue.
- **`multi-brand-portfolio`** — brand-wall'd dashboard with explicit "switch brand" gesture.
- **`cross-shop-rollup`** — chain Founder's surface, per-shop drill-down.
- **`cross-tenant-intelligence` (real insights)** — first peer-sets formed; insights begin once k≥10. Per ADR 0014 differential privacy.
- **`staff-promotion-flow`** — graduates an STA to ADM-D with scope.
- **`hiring-intake-surface`** — Owner-facing JD drafter + application reviewer.

### Migration tooling added
- ✅ Fresha API broker
- ✅ Square API broker
- ✅ Booksy CSV importer (graduates from "yes if demand" to standard).

### Pricing tiers opened

- ✅ Multi-brand pricing (€99/brand-shell + tier per shell).

### Integrations added

## Acceptance criteria for v1.5 ship

1. v1 acceptance criteria continue to hold.
2. Chair-rental host workflow has ≥3 paying C10 customers operating successfully for 30 days.
3. Cross-shop rollup tested with ≥2 paying C7 customers across multiple shops.
4. Multi-brand brand-wall verified: zero data leak across brand-shells in 30-day audit.
5. Cross-tenant intelligence: at least one peer-set has reached k≥10 and surfaced insights consumed by ≥3 tenants without privacy complaint.
6. UK design-partner cohort (≥3) onboarded.
7. Eval suite golden datasets expanded per new persona × cell at sizes per ADR 0016.

## v1.5 → v2 graduation criteria

- v1.5 holds SLOs for 90 days.
- ≥250 paying tenants total.
- Cross-tenant intelligence opt-in rate ≥35%.
- UK cohort NPS ≥40.
- Eval suite size per persona × locale meets v2 expansion target (+50% over v1.5).
- Founder + head of engineering + head of CS sign v2 RFC.
