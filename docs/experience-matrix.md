# Experience matrix

**Status:** F3 spine doc (2026-05-07). Single-table cheat sheet build/design references daily.

One row per *populated cell × primary modality* (filtered to v1-critical and v1-supported per F1's tagging).

Columns:
- **Cell** — persona × configuration × vertical
- **Modality** — primary surface
- **Default tab** — first thing they see
- **First-frame copy** — Liv's opening line
- **Ritual surface** — the daily/weekly thing they return for
- **Canonical alert** — the one notification that defines the relationship
- **Native moment** — the moment Liv earns the day
- **Promise** — one-line
- **Anti-list (compressed)** — what Liv refuses
- **Rung today/target** — Day 1 / Month 12
- **Hierarchy edges** — up / down
- **Competitor displaced** — primary

## How to read this matrix

- A row is a **cell × primary modality**. Some cells appear in multiple rows (Customer typology rows are six per vertical because the typology shifts the experience).
- "Default tab" is what they see when they open the app/dashboard fresh. Determines navigation defaults.
- "First-frame copy" is what Liv says — directly usable as draft copy in design.
- "Native moment" is the proof point — what makes this cell feel "I've never seen software do this."
- "Anti-list (compressed)" is the credibility — what Liv won't do that competitors might.
